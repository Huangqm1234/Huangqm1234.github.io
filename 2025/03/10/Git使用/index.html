同步update修改

git rebase意味着先把用户的修改放到一遍，把main最新的修改拿过来，在这个最新的修改基础之上，尝试把f-commit放上去。

此时可能会出现rebase conflict，这时候就需要手动选择。

相比较于merge，rebase是在主分支最新的提交上做出了修改。也就是说，相比较于merger会产生一个额外的merge commit，rebase会让提交更线性，更干净。但是公共分支不要使用。
